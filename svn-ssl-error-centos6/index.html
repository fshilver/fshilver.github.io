Svn Ssl Error Centos6

Heesung Yang

November 15, 2020 | 1 minute

troubleshoot

Issue

  • svn command SSL error with https svn server

    svn list https://10.10.10.10/svn/blurblur

    svn: OPTIONS of 'https://10.10.10.10/svn/blurblur': SSL handshake failed: SSL error: Key usage violation in certificate has been detected. (https://10.10.10.10)

Environment

  • CentOS 6.4

Solution

  • change the neon package’s compile option, rebuild and re-install

  1. set the repository to download source rpm

    vi /etc/yum.repo.d/source-rpm.repo

    [base-source]
name=CentOS-6.4 - Base source
baseurl=http://vault.centos.org/6.4/os/Source/
gpgcheck=1
gpgkey=http://vault.centos.org//RPM-GPG-KEY-CentOS-6
priority=1
enabled=1

[updates-source]
name=CentOS-6.4 - Updates Source
baseurl=http://vault.centos.org/6.4/updates/Source/
gpgcheck=1
gpgkey=http://mirror.centos.org/centos/RPM-GPG-KEY-CentOS-6
priority=1
enabled=1

  2. install yum-utils for yumdownloader command and install libproxy-devel, pakchois-devel for neon compile

    yum install -y yum-utils libproxy-devel pakchois-devel

  3. install neon source rpm

    yumdownloader --source neon
rpm -ivh neon-0.29.3-3.el6_4.src.rpm

  4. change the compile option ( with-ssl=gnu-tls -> with-ssl)

    cd ~/rpmbuild/
vi ~/rpmbuild/SPECS/neon.spec

    %configure --with-expat --enable-shared --disable-static \
        --enable-warnings --with-ca-bundle=%{cabundle} \
        --with-ssl --enable-threadsafe-ssl=posix \
        --with-libproxy

  5. build rpm and re-install

    rpmbuild -ba ~/rpmbuild/SPECS/neon.spec
rpm -e neon --nodeps
rpm -ivh ~/rpmbuild/RPMS/x86_64/neon-0.29.3-3.el6.x86_64.rpm
